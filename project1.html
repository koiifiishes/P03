GAME DESIGN : PET SALON

Role: Programmer and Lead Artist

Introduction

'Pet Salon' is a multimedia simulation created as a final project for IAT 265 (Multimedia Programming for Art and Design) in Spring 2022 at Simon Fraser University. The project consists of images, text, and music implemented using Java to create an interactive simulation for the user.

The Problem

I was tasked with creating an interactive simulation using Java that fulfilled a set of basic criteria, these included:

  • Generate perlin noise graphics
  • Use recursion to create fractal graphics
  • Use decorator patterns

Limitations

I was familiar with generating simple shapes using Java's Graphics/Graphics2D class, but creating the graphics like Perlin noise and fractal graphics required understanding how for-loops and recursive functions would affect the Graphics class.

We were taught factory classes for decorator patterns a week before the project's due date, so I did not have much knowledge of the topic prior. I had to learn how to apply them to my project within a small time frame.

Solution

Sketches

I begin my process by creating a list of items and tasks that the user can perform in 'Pet Salon'. This included: a light switch to turn on the lights, a soap dispenser to get soap, a switch to turn on the shower, a towel to dry the dog, decorative items to dress up the dog, etc. By creating a list, I can get a basic idea of the items I would need to draw for my simulation.

I came up with an initial sketch where a dog is sitting in the middle of a grooming table with all the items located around the it. After the initial sketch, I planned out the colours by colouring underneath my sketch roughly to make sure I had enough contrast between each element.

Perlin Noise

The Perlin noise graphics were generated using a for-loop in Java which generated noise in the form of randomized values. These values are then used to create many grey circles that overlap each other to create an effect that emulates the texture of soapy suds. To further add variation to user interactions, the more soap that the user picks up, the denser the Perlin noise suds will look.

Fractal Graphics

The fractal graphics require the use of recursion methods which means that a function in Java will call upon itself. Through the use of recursion, I was able to generate a plant graphic that branches out into smaller and smaller branches allowing it to resemble a life-like tree.

Decorator Patterns

Finally, Decorator patterns are used to dress up the dog towards the end of the simulation. The use of Decorator patterns allows the users to add new functionality (in this case: dress-up objects) to an object (the dog) without altering the objects original methods. In 'Pet Salon', the user can add ribbons, bows and jewelery to the dog by clicking the bins that hold the items and then clicking anywhere on the dog to apply the decorator.

Reflection

'Pet Salon' was a successful project because I was able to complete the project and all its requirements within the time that it was assigned.

One optional component for the project was include to creative elements that show that we went above and beyond the project requirements. By including more of these creative elements, we get awarded with more marks. I expanded my project from the project requirements by including elements like various different types of decorator objects (ribbons, bows and jewelery). And within these categories, the objects also have randomized designs: 3 ribbons, 3 bows and 2 jewelery. The variation in suds generated is another example of these creative elements.

If I were to do this project again, I would have included more animations into the simulation to make the simulation more life-like rather than a static environment that the user interacts with.
