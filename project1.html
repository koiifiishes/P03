GAME DESIGN : PET SALON

Role: Programmer and Lead Artist

'Pet Salon' is a multimedia simulation created as a final project for IAT 265 (Multimedia Programming for Art and Design) in Spring 2022 at Simon Fraser University. The project consists of images, text and music implemented using Java to create an interactive simulation for the user.

To begin the assignment, I had create a proposal with a short description of the project and outline some main coding elements that I plan to include. In 'Pet Salon', the user is guided through a process of operating a dog grooming salon. The user can follow the instructions through series of mouse clicks, mouse drags, animations, and more. My proceess began with creating a list of items and tasks that the user can preform in the simulation: light switches to turn on the lights, soap dispenser to get soap, switches to turn on the shower, towel to dry the dog, ribbons and bandana to dress-up the dog, etc. Based on this list, I started sketching out how I wanted these elements to be laid out in the simulation. From the user's perspective, the dog would be on a grooming table and all the items and tools around the dog. After the initial sketch, I decided to plan out the colours by colouring underneath my sketch roughly. I wanted to make sure all elements were distinct from one another so I had to create enough contrast between them. I used a mix of light and dark colours between background and foreground elements to push that distinction.

The project required a basic set of criteria to be integrated within the simulation, this included:

  • Generate perlin noise graphics
  • Use recursion to create fractal graphics
  • Use decorator patterns

The perlin noise graphics were generated using a for-loop in Java which generated noise in the form of randomized values. These values are then used to create many grey circles that overlap each other to create an effect that emulates the texture of soapy suds. To further add variation to user interactions, the more soap that the user picks up, the more dense the perlin noise suds will look.

The fractal graphics require the use of recursion methods which means that a function in Java will call upon itself. Through the use of recursion, I was able to generate a plant graphic that branches out into smaller and smaller branches allowing it to resemble a life-like tree (circled in red).

Finally, decorator patterns are used to dress up the dog towards the end of the simulation. The use of decorator patterns allows the users to add new functionality (in this case: dress-up objects) to an object (the dog) without altering the objects original methods. In 'Pet Salon', the user can add ribbons, bows and jewelery to the dog by clicking the bins that hold the items and then clicking anywhere on the dog to apply the decorator.

'Pet Salon' was a successful project because I was able to complete the project and all its requirements within the time that it was assigned. One optional component for the project was include to creative elements that show that we went above and beyond the project requirements. By including more of these creative elements, we get awarded with more marks. I expanded my project from the project requirements by including elements like various different types of decorator objects (ribbons, bows and jewelery). And within these categories, the objects also have randomized designs: 3 ribbons, 3 bows and 2 jewelery. The variation in suds generated is another example of these creative elements. If I were to do this project again, I would have included more animations into the simulation to make the simulation more life-like rather than a static environment that the user interacts with.
